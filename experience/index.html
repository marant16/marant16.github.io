Data Engineering & Intelligent Automation #

Project 1: High-Accuracy Invoice & Ticket Digitalization Pipeline #

A commercial firm was facing the risk of significant government fines due to errors in their manual data entry process. I created a system that could accurately digitalize their 3,000+ printed invoices and tickets.

  • Data Ingestion: Developed a simple mobile-friendly web interface allowing employees to upload photographs of documents directly from their phones.
  • Intelligent Document Processing: Architected a data extraction pipeline using an ensemble of OCR (Optical Character Recognition) engines and LMMs (large multimodal models) to maximize accuracy. This included a fallback system to flag ambiguous data for human verification.
  • Automated Validation: Reverse-engineered the national tax authority’s (SUNAT) web portal to build a service that automatically validated every digitalized invoice against the official government database.
  • Data Storage: Wrote the validated, structured data into a clean database for the client’s use.

Key Outcome: The system successfully processed the documents with an extremely low error rate, protecting the company from government fines.

Technologies Used: Python, LLM APIs, OCR Libraries, Web Scraping, SQL databases.

Project 2: Custom OCR for Handwritten Invoices in a Tax Fraud Defense Case #

A company accused of tax fraud struggled with their legal defense since they lacked a digital accounting workflow, I turned their 8,000+ handwritten documents into a digital database with high accuracy.

  • Data Annotation: Set up and managed a local instance of Label Studio (OSS) to coordinate a team of labelers, which allowed to collect training data for our custom models.
  • Image Preprocessing: Implemented a pipeline of image processing techniques to improve OCR accuracy.
  • Custom Model Development: Fine-tuned a Faster R-CNN object detection model to accurately segment and identify key fields (dates, amounts, company names) within the structure of the handwritten invoices.
  • Hybrid Extraction: Integrated the custom-trained models with external APIs and fallback mechanisms to create a resilient and accurate data extraction process.

Key Outcome: Developed a system capable of processing their irregular, handwritten documents with great accuracy.

Technologies Used: Python, PyTorch, OpenCV, Label Studio, Pandas, LMM APIs.

Project 3: Secure, Searchable Index of a 10GB Legal Case File #

A team of court-appointed accountants needed to make a report on the finances of a few individuals involved in a case with a large, unorganized court-file. I delivered a tool with ready to use search functionality across the entire file.

  • Data Strategy: Performed exploratory data analysis using vector embeddings and clustering to understand the document landscape and create an intelligent sampling strategy for labeling.
  • Process Optimization: Benchmarked various OCR methods on cost, speed, and accuracy to select the optimal approach.
  • Final Delivery: Processed and indexed the entire 10GB dataset. The final product was a ready-to-use solution delivered via Recoll (an open-source desktop search tool), allowing the client to instantly perform searches on the entire case file.

Key Outcome: Transformed a chaotic 10GB data dump into a usable record, allowing the legal team to find information effectively.

Technologies Used: Python, Sentence Transformers, Scikit-learn, OCR Libraries, Pandas, Recoll, Docker.
